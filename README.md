# 📚 Semantic Book Recommender — Gemini Edition

A semantic book recommendation system powered by **Google Gemini embeddings**, **LangChain**, **ChromaDB**, **Hugging Face Transformers**, and **Gradio**.
---

## ✨ Features

| Feature | Details |
|---|---|
| **Semantic search** | Natural-language queries matched against book descriptions using Gemini `text-embedding-004` vectors stored in ChromaDB |
| **Category filter** | Fiction / Non-Fiction labels generated by zero-shot classification (`facebook/bart-large-mnli`) |
| **Emotional tone sort** | Sort results by Joy, Fear, Sadness, Surprise, or Anger — scores extracted sentence-by-sentence using `j-hartmann/emotion-english-distilroberta-base` |
| **Gradio UI** | Clean Glass-themed gallery dashboard with 16 book cover results |

## 🚀 Quick Start

### 1. Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/semantic-book-recommender-gemini.git
cd semantic-book-recommender-gemini
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # macOS / Linux
# venv\Scripts\activate         # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Set up your Google API key

```bash
cp .env.example .env
```

Open `.env` and paste your key:
```
GOOGLE_API_KEY=AIza...
```

Get a **free** key at [aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey).

### 5. Download the dataset

Download the **7K Books Dataset** from Kaggle:
[kaggle.com/datasets/dylanjcastillo/7k-books-with-metadata](https://www.kaggle.com/datasets/dylanjcastillo/7k-books-with-metadata)

Place the CSV at `data/books.csv`.

### 6. Run the notebooks in order

```
notebooks/1-data-exploration.ipynb      → data/books_cleaned.csv
notebooks/2-vector-search.ipynb         → data/tagged_description.txt + data/chroma_db/
notebooks/3-text-classification.ipynb   → data/books_with_categories.csv
notebooks/4-sentiment-analysis.ipynb    → data/books_with_emotions.csv
```

> ⚠️ Notebooks 3 and 4 run Hugging Face models on CPU and can take **30–90 minutes** total. Run them once and the output CSVs are saved — you won't need to re-run them.

### 7. Launch the Gradio dashboard

```bash
python gradio-dashboard.py
```

Open your browser at **http://127.0.0.1:7860**

## 🧠 How It Works

```
User query (natural language)
        │
        ▼
Google Gemini text-embedding-004
(task_type="retrieval_query")
        │
        ▼
ChromaDB similarity search → top 50 candidate books
        │
        ▼
Parse ISBN13 from document text → filter main DataFrame
        │
        ├── Category filter (fiction / non-fiction)
        │
        ├── Tone sort (joy / fear / sadness / surprise / anger)
        │
        ▼
Top 16 books → Gradio Gallery
```

### Why `task_type` matters for Gemini embeddings

Gemini's `text-embedding-004` model accepts a `task_type` parameter that optimises the embedding for its purpose:

- `"retrieval_document"` — used when **indexing** documents into the vector store (notebook 2)
- `"retrieval_query"` — used when **searching** at query time (the dashboard)

This asymmetric embedding improves retrieval quality compared to using the same embedding for both.

## 📦 Tech Stack

| Layer | Library |
|---|---|
| Data processing | `pandas`, `numpy` |
| Zero-shot classification | `facebook/bart-large-mnli` via `transformers` |

| Vector store | `ChromaDB` via `langchain-chroma` |
| Orchestration | `langchain`, `langchain-community` |
| UI | `gradio` |
| Config | `python-dotenv` |
